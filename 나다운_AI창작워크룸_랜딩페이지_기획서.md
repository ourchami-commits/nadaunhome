## 1. 프로젝트 개요

### 1.1 브랜드 정의

| 항목 | 내용 |
|------|------|
| **브랜드명** | 나다운 AI활용 창작 워크룸 |
| **한 줄 정의** | AI를 몰라도, 나다운 콘텐츠와 새로운 가능성을 만들 수 있도록 돕는 창작형 교육 브랜드 |
| **운영 형태** | 온라인 클래스 + 기관·단체 오프라인 특강·워크숍 |
| **대표 가치** | "배움보다 완성, 기능보다 결과물" |
| **홈페이지 성격** | **개인 브랜딩형 홈페이지** — 강사/운영자의 전문성과 브랜드를 중심에 두는 1인 브랜드 사이트 |
| **구조 방식** | 원페이지(One-page) 기반으로 시작 → 향후 멀티페이지로 단계적 확장 |
| **홈페이지 목적** | 클래스 신청 전환 / 기관 문의 유도 / 브랜드 신뢰 구축 / 강사 개인 브랜드 아카이브 |

### 1.2 브랜드 핵심 가치 (5가지)

1. **쉬운 AI 활용** — AI를 몰라도, 초보도 시작할 수 있다
2. **내 이야기 → 결과물** — 경험과 이야기를 실제 콘텐츠로 연결한다
3. **성취와 새로운 역할** — 취미를 넘어 성취감과 사회적 역할로 확장한다
4. **결과물 중심** — 배움이 아닌 완성에 집중한다
5. **현장 친화적 교육** — 기관 현장에서도 적용 가능한 친절하고 실전적인 방식

### 1.3 홈페이지 확장 로드맵

**1단계 (현재 기획 범위) — 원페이지 브랜딩 홈**

원페이지 안에 브랜드·클래스·결과물·후기·문의를 모두 담아 빠르게 론칭한다.

**2단계 (향후 확장) — 멀티페이지 확장**

| 확장 페이지 | 목적 |
|-------------|------|
| 프로그램 상세 페이지 | 클래스별 커리큘럼·수강료·일정 상세 안내 |
| 기관 강의 안내 페이지 | 기관·단체 대상 특강 소개 + 제안서 요청 폼 |
| 결과물 상세 페이지 | 포트폴리오 갤러리 + 작품별 스토리 |
| PDF 다운로드 페이지 | 클래스 소개서·커리큘럼 PDF 리드 수집 |

> 개발 시 Next.js App Router 구조로 설계하여 2단계 확장 시 추가 비용 최소화

### 1.4 개발 목표

- **1차 목표:** 개인 수강 신청 전환율 **5% 이상** 달성
- **2차 목표:** 기관·단체 특강 문의(리드) 월 **10건 이상** 확보
- **3차 목표:** "AI = 어렵다"는 인식 장벽 해소 → 브랜드 신뢰 기반 구축

## 2. 타겟 사용자 분석

### 2.1 1차 타겟 — 개인 수강생

| 구분 | 내용 |
|------|------|
| **연령** | 40~60대 (디지털 친숙도 낮으나 창작 욕구 높음) |
| **직업·역할** | 주부, 은퇴 후 새 출발 준비자, 프리랜서, 육아 중 창작 희망자 |
| **성향** | AI가 어렵다고 느끼지만 내 이야기를 뭔가로 남기고 싶은 사람 |
| **페인포인트** | "AI는 나랑 상관없는 것 같아요" / "내가 할 수 있을지 모르겠어요" |
| **목표** | 내 이야기로 그림책·영상·콘텐츠를 만들어 완성하고 싶다 |

### 2.2 2차 타겟 — 기관·단체 담당자

| 구분 | 내용 |
|------|------|
| **유형** | 복지관, 문화센터, 도서관, 평생교육원, 사회적기업 |
| **담당자** | 프로그램 기획 담당자, 교육팀장 |
| **니즈** | 어르신·중장년 대상 AI 교육 프로그램을 쉽고 완성도 있게 운영하고 싶다 |
| **페인포인트** | 기술 중심 AI 강사는 많은데, 창작 결과물까지 연결하는 곳이 없다 |

### 2.3 페르소나 예시

> **[개인] 박○○ (55세, 자녀 독립 후 창작 도전 준비 중)**
> 평생 아이들 책 읽어주는 게 좋았는데, 내가 직접 그림책을 만들 수 있으면 얼마나 좋을까 싶었어요.
> AI를 쓰면 된다는데 뭔지도 모르고, 나 같은 사람도 할 수 있는지 모르겠어요.

> **[기관] 이○○ (문화센터 프로그램 담당자)**
> AI 교육 요청이 오는데, 기술 설명만 하는 강사는 싫어요.
> 수강생들이 뭔가 직접 만들어 가져갈 수 있는 프로그램이 필요해요.

## 4. 제공 서비스 구성

### 4.1 클래스 라인업

**운영 구조:** 모든 클래스는 **6주 정규 과정**과 **2주 챌린지** 두 트랙으로 운영됩니다.

```
각 클래스
├── 6주 정규 과정  — 처음부터 끝까지 완성하는 전체 과정
└── 2주 챌린지    — 핵심만 압축해 첫 결과물을 빠르게 완성
```

| # | 클래스명 | 6주 정규 결과물 | 2주 챌린지 결과물 | 대상 |
|---|----------|-----------------|-------------------|------|
| 1 | **AI활용 기초 클래스** | AI 도구 전반 활용 능력 | AI 도구 기초 체험 | AI 처음 접하는 누구나 |
| 2 | **AI활용 시 그림책 출판 클래스** | 시 기반 그림책 완성 | 시 1편 + AI 이미지 1장 | 글쓰기·시 관심자 |
| 3 | **AI활용 그림책 출판 클래스** | 스토리 그림책 완성·출판 | 그림책 1~2페이지 완성 | 이야기 있는 누구나 |
| 4 | **AI활용 영상제작 클래스** | 숏폼/소개 영상 완성 | 30초 AI 영상 1편 | 영상 초보자 |
| 5 | **기관·단체 AI활용 특강·워크숍** | 맞춤형 교육 프로그램 | 맞춤 협의 | 복지관·도서관·문화센터 등 |

### 4.2 클래스별 핵심 포인트

**2주 챌린지 공통 개념 (전 클래스 적용)**
- 각 클래스의 핵심 경험을 2주로 압축한 단기 트랙
- 6주 정규 과정이 부담스러운 분의 첫 진입 경로
- 챌린지 수료 후 정규 6주 과정 연결 시 할인 혜택 제공 가능
- 빠른 성취감 → 정규 과정 전환율 향상 목적

**AI활용 기초 클래스**
- AI가 처음인 분들을 위한 진입 클래스
- 스마트폰·PC 기초 활용 수준에서 시작
- 6주: AI 도구 전반 기초 완성 / 2주 챌린지: AI 핵심 도구 1가지 체험

**AI활용 시 그림책 / 그림책 출판 클래스** (핵심 플래그십)
- 내 글·이야기 → AI 이미지 생성 → 그림책 편집 → 출판
- "내가 만든 책"이라는 성취감이 핵심, 소량 인쇄·전자출판 연결 가능
- 6주: 완성된 그림책 출판 / 2주 챌린지: 그림책 1~2페이지 완성

**AI활용 영상제작 클래스**
- 스마트폰으로도 가능한 AI 영상 제작
- 6주: 숏폼/소개 영상 완성 / 2주 챌린지: 30초 AI 영상 1편 완성

**기관·단체 특강 및 워크숍**
- 수강 인원·시간·주제 맞춤 설계
- 1회 특강 / 정규 시리즈 모두 가능
- 문의 후 제안서 제공

### 섹션 00 — 글로벌 네비게이션 바 (상단 고정)

**목적:** 페이지 어느 위치에서든 빠른 이동 + 주요 CTA 상시 노출

#### 레이아웃 구조

```
[ 로고 (왼쪽) ]                    [ 메뉴 + CTA 버튼 (오른쪽) ]
나다운 AI창작 워크룸    클래스 소개  기관 특강  후기  FAQ  [신청하기]
```

#### 구성 요소 상세

| 위치 | 요소 | 내용 |
|------|------|------|
| **왼쪽** | 브랜드 로고 | 텍스트 로고 또는 이미지 로고 / 클릭 시 페이지 최상단 이동 |
| **오른쪽 — 메뉴** | 클래스 소개 | 섹션 04 (클래스 소개)로 앵커 이동 |
| **오른쪽 — 메뉴** | 기관 특강 | 섹션 08 (문의하기)로 앵커 이동 |
| **오른쪽 — 메뉴** | 수강 후기 | 섹션 06 (후기)로 앵커 이동 |
| **오른쪽 — 메뉴** | FAQ | 섹션 09 (FAQ)로 앵커 이동 |
| **오른쪽 — CTA** | 신청하기 버튼 | Primary 색상 버튼 / 섹션 04 클래스 소개로 스크롤 |

#### 동작 스펙

| 상태 | 동작 |
|------|------|
| **기본 (최상단)** | 배경 투명 또는 크림 색상, 로고 + 메뉴 + CTA 표시 |
| **스크롤 후** | 배경 흰색 + 하단 shadow 추가 (sticky 고정 유지) |
| **모바일** | 로고 (왼쪽) + 햄버거 메뉴 아이콘 (오른쪽) |
| **모바일 메뉴 오픈** | 전체 화면 드로어(Drawer) 또는 드롭다운 — 메뉴 항목 + 신청하기 버튼 |
| **현재 섹션 표시** | 스크롤 위치에 따라 해당 메뉴 항목 활성화 (active 스타일) |

#### 디자인 스펙

| 속성 | 값 |
|------|-----|
| 높이 | 64px (데스크탑) / 56px (모바일) |
| 배경 | 스크롤 전: `transparent` / 스크롤 후: `#FFFFFF` + `box-shadow` |
| z-index | `999` (항상 최상위) |
| position | `sticky top-0` |
| 메뉴 텍스트 색상 | `#1C1C1A` / hover 시 Primary 색상(`#3D6B4F`) |
| CTA 버튼 | Primary 색상(`#3D6B4F`) 배경, 흰색 텍스트, border-radius 8px |
| 전환 효과 | 배경 색상 변화 `transition: 0.3s ease` |

### 섹션 02 — 공감 & 문제 제기

**목적:** "맞아, 나도 이런 생각 했었어" 공감 형성

**카드 형식 3가지 고민:**

1. **"AI는 젊은 사람들 것 아닌가요?"**
   → 나이와 상관없이, 스마트폰을 쓸 수 있으면 시작할 수 있어요

2. **"배우고 싶은데 뭔가를 만들 수 있을지 모르겠어요"**
   → 이곳은 배우는 게 목표가 아니에요. 완성하는 게 목표입니다

3. **"기관에서 AI 교육을 해야 하는데 어디에 맡겨야 할지 몰라요"**

### 섹션 04 — 클래스 소개

  - 주차별 커리큘럼
  - 수강료 (각 트랙별)
- CTA: "신청하기" → 클래스별 외부 링크로 이동 (새 탭 열기)
  - 구글폼 또는 온라인 강의 플랫폼(라이브클래스 등) 링크 연결

| 이벤트명 | 트리거 |
|----------|--------|
| `cta_click_hero` | Hero CTA 버튼 클릭 |

## 11. 개발 일정

### 11.1 개발 단계 구조

```
1단계: 원페이지 브랜딩 홈 론칭  →  2단계: 멀티페이지 확장
(현재 기획 범위)                      (운영 후 순차 추가)
```

### 11.2 1단계 마일스톤 — 원페이지 론칭 (6주)

| 단계 | 기간 | 주요 작업 |
|------|------|-----------|
| **기획 확정** | Week 1 | 카피 최종 확정, 결과물 이미지 수집, 후기 수집 |
| **디자인 시안** | Week 2 | 와이어프레임 → Figma 목업 (모바일 + 데스크탑) |
| **개발 1차** | Week 3~4 | 퍼블리싱, 컴포넌트 구현, 폼 연동, 확장 대비 URL 구조 설계 |
| **검수 & QA** | Week 5 | 반응형 테스트, 폼 동작, 브라우저 호환성 |
| **론칭** | Week 6 | 도메인 연결, GA4 설정, SNS 채널 공유 시작 |

### 11.3 2단계 마일스톤 — 멀티페이지 확장 (운영 후 순차)

| 확장 페이지 | 추가 조건 | 예상 작업 규모 |
|-------------|-----------|----------------|
| 프로그램 상세 페이지 (`/classes/[slug]`) | 클래스별 상세 커리큘럼 확정 후 | 소 (원페이지 콘텐츠 분리) |
| 기관 강의 안내 페이지 (`/institution`) | 기관 문의 증가 시 | 소 |
| 결과물 갤러리 (`/portfolio`) | 결과물 10건 이상 누적 후 | 중 |
| PDF 다운로드 페이지 (`/downloads`) | 리드 수집 전략 강화 시 | 소 |

> 1단계 개발 시 Next.js App Router 디렉토리 구조를 확장 기준으로 미리 잡아두면 2단계 추가 시 비용·시간 최소화 가능
